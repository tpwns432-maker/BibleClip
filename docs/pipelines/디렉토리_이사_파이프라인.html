BibleClip · Repository Layout

디렉토리 이사 파이프라인

이름은 BibleClip이 됐는데 폴더는 아직 AutoBible 시절 그대로입니다. 루트에 쌓인 19MB 잡동사니를 치우고, 코드·데이터·빌드·문서를 제자리로 옮깁니다. GitHub 저장소 이름(AutoBible)·사용자 데이터는 그대로 유지.

루트 파일 20+ → 정돈 삭제 대상 ≈ 19.5 MB 빌드 참조 3곳 동기화

0먼저: 절대 건드리면 안 되는 것

이사 중 이 4가지가 깨지면 사용자 데이터 손실 또는 업데이트 단절. 모든 단계에서 사수합니다.

icon 이동은 선택사항: 아이콘을 assets/icons/로 옮기면 CI(--icon=icon.ico, Copy-Item)·spec·build_mac.sh 3곳을 같이 고쳐야 합니다. 위험을 더 줄이려면 icon 파일은 루트에 그대로 두는 것을 권장(CI 무수정). 나머지 정리만으로도 루트는 충분히 깔끔해집니다.

4같이 고쳐야 할 빌드 참조

파일을 옮기면 이 참조들이 깨집니다. 이동과 한 커밋에서 함께 수정해야 CI가 안 끊깁니다.

파일현재 참조이동 시 조치
.github/workflows/build.yml --icon=icon.ico / icon.icns
Copy-Item bible_versions/...
--name BibleClip autobible.py		 autobible.py·데이터 폴더는 루트 유지 → 무수정. icon만 옮기면 icon 경로 갱신
build/BibleClip.spec ['autobible.py'] / name='AutoBible' name→BibleClip; spec를 build/로 옮기면 pathex=['..'] 또는 루트에서 실행
build/build_mac.sh cd "$(dirname "$0")"
cp bible_versions/* ...		 build/ 하위로 가면 cd "$(dirname "$0")/.." 로 루트 이동 후 기존 상대경로 유지
.gitignore 개별 항목 새 폴더(docs/pipelines/ 등) 추적 여부 정리, 삭제물 항목 제거
핵심 보호 장치: 가장 깨지기 쉬운 autobible.py·bible_versions/·original_lang/를 루트에 고정했기 때문에, 실제로 손대는 빌드 참조는 (icon을 안 옮기면) spec 이름변경 한 곳뿐입니다.

5실행 단계

git 이력 보존을 위해 git mv 사용. 단계마다 검증 게이트.

  • Phase 0 · 안전망

    브랜치 + 데이터 무결성 확인

    git switch -c chore/repo-reorg. original_lang/ 3개 파일 존재 확인(BethlehemWin 삭제 전제). 현재 v1.5.0 정상 실행을 기준점으로 기록.

    ✓ 앱 실행 + 원어 패널 동작 확인
  • Phase 1 · 비우기

    잡동사니 삭제

    AutoBible-mac-source.zip·BethlehemWin/·image.jpg·update_check.log 삭제 (대부분 gitignore라 작업트리에서만 제거). 디스크 19.5MB 회수.

    ✓ git status 깨끗 + 앱 여전히 실행
  • Phase 2 · 폴더 골격

    docs/ · build/ · assets/ 생성 후 git mv

    문서·빌드스크립트·(선택)아이콘을 git mv로 이동(이력 보존). bible_versions/·original_lang/·autobible.py는 건드리지 않음.

    ✓ git log --follow 로 이력 추적 확인
  • Phase 3 · 참조 동기화

    spec 이름변경 + (필요시) 경로 갱신

    AutoBible.spec→build/BibleClip.spec, 내부 name 수정. build_mac.sh의 cd 보정. icon을 옮겼다면 CI·spec·sh의 icon 경로 일괄 수정.

    ✓ 로컬 PyInstaller 빌드 성공
  • Phase 4 · 문서 갱신

    README 작성 + 경로 안내 수정

    루트에 README.md 신설(빌드/실행/구조 요약). BUILD_MAC.md의 경로 참조를 새 위치로. CHANGELOG에 "저장소 구조 정리" 기록.

    ✓ 문서 링크·경로 유효성 확인
  • Phase 5 · CI 검증

    main 머지 → 다음 태그에서 빌드 확인

    구조 변경은 기능 무관 → 단독 커밋으로 머지. 다음 버전 태그 푸시 시 Windows/macOS 빌드가 새 경로에서도 green인지 확인.

    ✓ CI 3 잡 green + 자산 3개

6모듈화와의 순서

두 작업은 독립적이지만, 이사 → 모듈화 순서를 권장합니다.

순서작업이유
1차디렉토리 이사 (이 문서)잡동사니 먼저 비우면 모듈화 시 탐색이 깔끔. 위험 낮고 빠름
2차코드 모듈화 (별도 문서)정리된 루트 위에 bibleclip/ 패키지 생성 → 충돌 없음

두 작업 모두 autobible.py를 루트 엔트리로 유지하므로 CI·릴리스 파이프라인은 내내 끊기지 않습니다. 각각 별도 브랜치·별도 버전으로 진행 가능.
