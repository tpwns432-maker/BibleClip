3통합 실행 순서

GROUP A · 정리 (위험 ↓ · 워밍업)
  • Phase 0 · 안전망

    브랜치 + 베이스라인 이모청

    git switch -c refactor/unify. v1.5.0 정상 실행 확인. original_lang/ 3파일 존재 확인. 현재 autobible_settings.json 값 기록(이전 검증용).
    ✓ 앱 실행 + 원어 패널 + 설정값 캡처
  • Phase 1 · 비우기

    잡동사니 삭제 (≈19.5MB) 이청

    AutoBible-mac-source.zip · BethlehemWin/ · image.jpg · update_check.log 삭제. 대부분 gitignore라 작업트리에서만 제거.
    ✓ git status 정상 + 앱 여전히 실행
  • Phase 2 · 폴더 골격

    docs/ · build/ 생성 후 git mv 이

    문서(CHANGELOG·BUILD_MAC·리포트·파이프라인 html)→docs/, 빌드스크립트→build/를 git mv(이력 보존). 데이터 폴더·엔트리·아이콘은 루트 유지(빌드 참조는 Phase 8에서 일괄).
    ✓ git log --follow 이력 추적 OK
GROUP B · 무손실 선작업
  • Phase 3 · 설정 마이그레이션

    설정 파일명 전환 슈팅 주입 청

    bibleclip_settings.json으로 읽고 쓰되, 없으면 옛 autobible_settings.json을 1회 승계. 옛 파일은 남겨 롤백 호환. 파일 어디서도 안 깨지게 가장 먼저.
    ✓ 옛 설정(버전순서·위치) 그대로 복원
GROUP C · 모듈화 (핵심)
  • Phase 4 · 잎 모듈

    bibleclip/ 패키지 + config·constants·theme·text_utils 모

    의존성 없는 순수 모듈부터 분리. autobible.py 상단을 import로 교체, 나머지는 그대로.
    ✓ 실행 + 본문 + 다크모드 토글
  • Phase 5 · 데이터·코어

    data/ (BibleDB·원어) · core/ (Engine·Formatter) 모

    백엔드 클래스 이동. 경로 로직(config) 옮긴 뒤 데이터 로딩 재확인 필수.
    ✓ 성경 로드 + 원어 패널 + #검색
  • Phase 6 · UI 믹스인

    ui/ 7개 믹스인 + app.py 조립 모

    Theme→Monitor→Search→Settings→Lexicon→Viewer→Update 순(작은 것부터). 추출 1개당 실행 1회.
    ✓ 믹스인 7회 각각 검증
GROUP D · 네이밍 flip + 빌드
  • Phase 7 · 내부 흔적

    tmpdir 접두사·주석 등 코드 내부 이름 정리 청

    bibleclip_update_ 등. 유지 대상은 손대지 않음 — GITHUB_REPO="AutoBible", macOS 옛-번들 호환 로직, CHANGELOG 과거 기록.
    ✓ 업데이트 흐름·클립보드 정상
  • Phase 8 · 엔트리 + 빌드 동기화 (단일 패스)

    슈팅 개명 + spec·CI·build_mac.sh 일괄 이모청

    autobible.py(이제 슈팅)→bibleclip.py 또는 python -m bibleclip. spec name='BibleClip'·엔트리, CI 엔트리·--collect-submodules bibleclip, build_mac.sh cd "..". 세 계획이 겹쳤던 빌드 수정을 여기서 한 번에.
    ✓ 로컬 PyInstaller 빌드 + 실행 성공
GROUP E · 마무리
  • Phase 9 · 문서

    README 신설 + 경로·CHANGELOG 갱신 이청

    README(구조·빌드·실행). BUILD_MAC 경로 갱신. CHANGELOG에 "내부 구조 정리(모듈화+네이밍, 기능 동일)" 추가, 과거 기록 보존.
    ✓ 문서 명령 그대로 실행 가능
  • Phase 10 · 릴리스

    v1.6.0 범프 → 태그 → CI 이모청

    단일 릴리스. Windows/macOS zip·dmg 확인. 최종 관문: 기존 사용자 인앱 업데이트 후 설정 보존.
    ✓ CI 3잡 green + 자산 3개 + 설정 보존
