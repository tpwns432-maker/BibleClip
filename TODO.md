# BibleClip v1.1.4 Master UI/UX, Animation & Multi-Window Refinement Specifications

본 문서는 BibleClip v1.1.4 패치에서 진행할 유저 편의성 극대화, 다중 윈도우 인터랙션 확장, 장바구니 영속성 확보, 그리고 자식 창 프로세스 좀비 버그 픽스를 정의한 마스터 디렉티브 명세서입니다. 본 명세서의 사양을 엄격히 준수하여 코딩을 진행하고, 완필 후 지명도 파일을 갱신하십시오.

---

## 🚀 Part 1. Multi-Window Expansion & Integration (다중 창 확장 및 연동)

### ✨ [FEAT-07] 장바구니 독립 윈도우(팝업 창) 분리 기능 및 메인 뷰어 점프 연동
- **기획 배경:** `F11` 프리젠테이션 모드 가동 시 메인 화면이 전체화면으로 전환되므로, 우측 레일 패널에 존재하는 장바구니를 동시에 볼 수 없어 설교 진행 제어가 불가능함. 장바구니를 독립 창으로 분리하여 듀얼 모니터 환경 등에서 멀티 태스킹이 가능하도록 개선함.
- **구현 사양:**
  - 우측 장바구니 레일 패널 상단에 **[새 창으로 열기 (Pop-out)]** 버튼을 신설할 것.
  - 버튼 클릭 시, 원전 분해 사전 창과 동일하게 `pywebview.create_window`를 경유하여 독립된 **장바구니 전용 자식 윈도우**를 커스텀 타이틀 및 정제된 미니멀 레이아웃으로 생성할 것.
  - **크로스 윈도우 인터랙션 (핵심):** 분리된 장바구니 창 내부 리스트에서 특정 성구를 클릭했을 때, IPC 브릿지를 경유하여 **메인 워크스페이스(F11 전체화면 상태 포함)의 메인 성경 뷰어 카드가 해당 성구 장/절 위치로 즉각 포커스 점프(`goToRef`)**하도록 이벤트 파이프라인을 연동할 것.

## 💾 Part 2. Data Persistence & Process Lifecycle (데이터 영속성 및 버그 픽스)

### 🚨 [BUG-SYS] 메인 윈도우 종료 시 자식 윈도우(원전 분해 사전 등) 좀비 잔존 현상 수정
- **현상:** 사용자가 메인 프로그램을 종료(X 버튼 클릭)했음에도 불구하고, 우클릭으로 열어두었던 원전 분해 사전 독립 창 등이 함께 닫히지 않고 바탕화면에 흉하게 남아있는 프로세스 생명주기 이탈 현상 발생.
- **해결 방안:** 백엔드 `app.py` 또는 `api.py` 단에서 메인 창의 `closing` 또는 `destroyed` 이벤트를 인터셉트하여, 현재 생명주기 관리 스택에 등록된 모든 자식 윈도우 인스턴스(사전 창, 신설된 장바구니 창 등)를 루프를 돌며 명시적으로 `destroy()` 처리 후 프로세스를 안전하게 전면 종료(Exit)시킬 것.

### ✨ [FEAT-08] 프로그램 재구동 시 장바구니 데이터 자동 복구 (영속성 패치)
- **현상:** 장바구니에 열심히 성구를 담아두었다가 프로그램을 껐다 켜면 리스트가 완전히 초기화되어 데이터가 소실됨.
- **구현 사양:** 장바구니 상태 데이터가 변경될 때마다 브라우저 로컬 스토리지(`localStorage.setItem('bibleclip_cart', ...)`) 또는 로컬 JSON 파일에 실시간 스냅샷을 영속성 저장하고, 앱 부팅 및 초기화(`get_initial_data`) 시 이를 역직렬화하여 기존 장바구니 상태를 완벽하게 복구해 낼 것. 사역자들의 소중한 설교 대지 상태를 상시 보존할 것.

## 🎨 Part 3. Core UI/UX Refinement (기존 UI/UX 및 애니메이션 벼림)

### 1. 메인 화면 글로벌 언어 토글 제거 및 원전 분해 카드 소스 다양화
- 메인 화면 상단 영역에 존재하는 모호한 한/영 로케일 토글 버튼을 완전히 제거(Deprecate)할 것. 전체 시스템 UI는 환경설정(Settings)의 언어 설정을 고정 순위로 추종함.
- **[원전 분해]** 카드 헤더 단에 소형 드롭다운 선택창(Select UI)을 배치하여, 사용자가 보유한 원어 데이터 소스 리스트(`Default(설정추종)`, `개역한글S`, `KJV+` 등)를 독립적으로 선택 및 전환할 수 있도록 구조를 유연화할 것.

### 2. 묵상 노트 아이콘 렌더링 위치 이동 (본문 가독성 확보)
- 본문 카드의 구절 렌더링 루틴을 변경하여, 노트가 존재하는 구절의 아이콘 표기 위치를 절의 맨 앞이 아닌 **[텍스트 본문 문장이 끝나는 최종 지점 뒤]**로 완전히 이동시킬 것. (예: `~천지를 창조하시니라. [📝]`)
- **[F11 연동]:** 사용자가 `F11` 프리젠테이션 모드를 트리거(`presentToggle` 활성화)할 때, 본문 내의 모든 묵상 노트 알림 아이콘 엘리먼트들을 CSS 스타일 규칙을 통해 무조건 `display: none !important;` 처리하여 순수 성경 텍스트만 깨끗하게 표출되도록 필터링할 것.

### 3. 네비게이션 용어 정립 및 사이드 레일 패널 애니메이션 전면 재수술
- 용어 정의: 왼쪽 아이콘 바는 `사이드바/네비게이션 레일`, 우측에서 개폐되는 슬라이딩 영역은 `레일 패널/사이드 패널`로 명칭 인덱스를 통일함.
- CSS hardware-accelerated `transform: translateX()` 및 `transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1)` 규칙을 적용하여, 패널이 실제 우측 화면 경계 너머에서 부드럽게 스르륵 밀려나오고 들어가는 진짜 레일 무빙을 구현할 것.
- **Light Dismiss:** 레일 패널이 활성화되어 있을 때, 패널 영역이 아닌 메인 워크스페이스 성경 보기 영역을 클릭하면 패널이 개방될 때와 동일하게 부드러운 아웃 애니메이션을 타며 벽 뒤로 들어가 닫히도록 예외 처리 스크립트를 보강할 것.

### 4. 장바구니 드래그 앤 드롭 (Drag & Drop) 메커니즘 전면 재구축
- HTML5 Native Drag boundaries 이벤트를 원점에서 재점검하여 사용자가 칩을 잡고 움직이는 동안 타깃 칩들이 유기적으로 트랜지션되며 자리를 비켜주는 반응형 무빙(진짜 실시간 위치 변경 효과)을 완성하고, 드롭 시 배후 데이터 정렬 무결성을 확실하게 확보할 것.

### 5. 묵상 노트 파일 저장 출력 시 구절별 구분선(`---`) 주입
- 슬라이딩 노트 레일 패널에서 사용자가 묵상 메모를 텍스트 파일로 익스포트(Export)할 때, 긴 문장이나 줄바꿈(New line) 메모가 섞여 구절 경계가 모호해지는 것을 방지하기 위해 구절 데이터 덩어리 사이에 명확한 구분선 문자열(`--------------------------------------------------`)을 강제 피딩하여 출력 가독성을 높일 것.
