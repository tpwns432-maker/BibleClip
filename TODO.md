# TODO: v1.0.6 — 글로벌 i18n 기반 + 프로페셔널 방송 송출 패치

## 📌 개요
- **이번 패치의 1순위는 다국어(i18n) 뼈대**다. 앞으로 추가되는 모든 UI 문자열이
  이 인프라 위에 올라타야 이중작업이 없다. 그래서 기능 개발보다 **먼저** 깐다.
- 그 위에 방송 송출·실무 생산성 기능(장바구니, F11 전체화면+F2 팝업, 폰트 파이프라인)을
  얹되, **새로 만드는 문자열은 처음부터 `t("key")` 방식**으로 작성한다.

### ⚙️ 아키텍처 확정 사실 (작업 전 필독)
- **이 앱의 배포 UI는 웹 프론트엔드다.** (`web/index.html` + `web/js/*.js` + `web/css/`,
  pywebview/WebView2 렌더). `bibleclip/ui/*.py`(CustomTkinter)는 레거시이며 i18n 대상 아님.
- 따라서 i18n은 **파이썬이 아니라 프론트엔드(JS/HTML)에서** 처리한다.
  (초기 구상의 `bibleclip/i18n.py`는 폐기 — 화면 문자열은 파이썬에 없다.)
- 웹 DOM은 실시간 재번역이 가능하고 네이티브 크래시와 무관하므로,
  **언어 전환에 "앱 재시작"이 불필요하다.** 즉시·무크래시 전환으로 간다.

- 카드 전체화면 및 팝업 중첩 시 에러 여부 브라우저 콘솔 검증.

  (**`web/locales/*.json` 동봉 여부 포함**).

### [NEW] 앱 내부 사용자 정의 약칭 추가 UI (역본별 책이름 별칭)
- **개념:** JSON 직접 편집 대신, 프로그램 안에서 사용자가 *역본별 책이름 약칭*을 직접
  추가·관리. (백엔드는 v1.0.6에서 만든 `bible_versions/aliases_override.json` 머지
  메커니즘을 그대로 사용 — UI가 이 파일에 써넣는 형태.)
- **⚠️ 필수 검증 — 숫자 약칭 차단:** 약칭에 숫자가 들어가 **장 번호와 충돌**하는 입력은
  거부해야 한다. 예: `요1`·`요2`(요한1서식), `벧전`을 `벧1`로 — 파서가 `요1 5:4`를
  "요1 책 5:4" vs "요 책 15:4"로 구분 불가. **한국어 숫자 책을 순수 한글(요일/요이/요삼)
  로만 받는 이유와 동일.** → 약칭이 `[숫자]`를 포함하거나 챕터 파싱과 모호해지면 막을 것.
- **기반:** 별칭맵은 영어 패턴 경로에만 주입돼 있음(앞 숫자 안전). 한국어/기타 언어
  *참조 파싱* 확장(정규식·스크립트별 숫자)은 별도 과제.
