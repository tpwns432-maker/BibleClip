# BibleClip — High-Speed Scripture Clipboard Controller for Everyday Christians
### 🇰🇷 일상 속 말씀 묵상과 인용을 위한 가장 빠른 스마트 클립보드 가속기

BibleClip is a high-performance desktop utility explicitly engineered to **automate and link the OS System Clipboard (`Ctrl+C` / `Ctrl+V`)** for everyday Christians, faith-based bloggers, authors, and digital ministry creators worldwide.

The core identity of BibleClip is an **Advanced Clipboard Accelerator**. It completely eliminates manual app-switching, browser searching, and tedious mouse-dragging, allowing you to instantly fetch, format, and share Bible verses directly through your system clipboard memory.

BibleClip은 매일 말씀을 묵상하고, 신앙 블로그를 쓰며, SNS나 단톡방으로 말씀을 공유하는 전 세계 기독교인들을 위해 설계된 고성능 데스크톱 유틸리티입니다. 성경을 찾기 위해 매번 인터넷 창을 켜고 마우스로 긁어 복사하는 번거로움을 완전히 지우고, 오직 시스템 클립보드(`Ctrl+C` / `Ctrl+V`)의 자동화와 고속 제어를 통해 일상 속 신앙 생활의 생산성을 극대화합니다.

## ⚡ Core Mechanism & Real-World Use Cases
### 🇰🇷 핵심 메커니즘 및 실제 활용 예시

Every feature in BibleClip is organically engineered to achieve one goal: **Perfect, Zero-Latency Clipboard Overwrite.**
BibleClip의 모든 기능은 오직 하나의 목표를 향해 있습니다: **"화면 전환 없는 완벽하고 즉각적인 클립보드 자동 전환."**

### 1. The Seamless `Ctrl+C` ➡️ `Ctrl+V` Quick Citation Loop
- **English:** While writing a faith-based blog post, personal journal, or theological paper, you no longer need to open a browser tab to search for verses. Simply type a shorthand scripture reference (e.g., `Rom 8:28` or `Gen 1:1`) directly inside your word processor (Notion, MS Word, Wordpad) and press **`Ctrl+C`**. BibleClip instantly captures this reference in the background, fetches the full corresponding verse, and overwrites your clipboard. You can immediately hit **`Ctrl+V`** to paste the complete scripture without ever leaving your writing window.
- **한국어 (앱 전환 없는 1초 말씀 인용):** 신앙 블로그 글을 쓰거나, 노션/워드에 묵상 기록을 남길 때 성경을 찾으려고 인터넷 창을 켤 필요가 없습니다. 내가 글을 쓰던 창 제자리에서 `롬 8:28` 혹은 `창 1:1`과 같은 구절 약어를 타이핑하고 블록을 잡아 **`Ctrl+C`**를 누르면 끝입니다. BibleClip이 백그라운드에서 0.01초 만에 이를 낚아채서 성경 본문 전체로 클립보드를 갈아끼워 줍니다. 유저는 화면 전환 없이 그 자리에서 바로 **`Ctrl+V`**만 누르면 완벽한 성경 구절 인용문이 텍스트로 주입됩니다.

### 💡 Real-World Example (How It Works)
- **The Trigger (`Ctrl+C`):** The user highlights a scripture reference (`John 3:16`) in their digital study notes and presses `Ctrl+C`.
- **The Interception:** BibleClip captures the token in milliseconds, executes an ultra-fast local database query, and overwrites the clipboard with the full verse text.
- **The Dispatch (`Ctrl+V`):** The user immediately presses `Ctrl+V` in their KakaoTalk chat window, blog editor, or presentation slide. The full text is generated instantly on screen without any manual copy lag.

### 2. Instant HUD Clipboard Overwrite via F2 Pop-up (v1.0.6)
- **English:** Want to share a sudden, uplifting verse with a friend on KakaoTalk, WhatsApp, or Slack? Simply press **`F2`** from any screen to summon a compact Heads-Up Display (HUD) input overlay. Type a keyword (e.g., "Love", "Grace", "Peace") and hit `Enter`. BibleClip's custom intelligent engine immediately searches the database, auto-copies the highest-scoring matching verse into your system clipboard, and dismisses the HUD. Hit **`Ctrl+V`** in your chat box to send it instantly.
- **한국어 (F2 단축키 퀵 카톡 공유):** 카카오톡이나 SNS로 지인들에게 은혜로운 말씀 구절을 빠르게 보내고 싶을 때, 컴퓨터를 하다가 화면 아무 데서나 **`F2`**를 딸깍 누릅니다. 화면 중앙에 슬림하게 뜨는 퀵 검색창(HUD)에 원하는 키워드(예: `사랑`, `위로`, `감사`)를 치고 엔터를 누르면, 가장 적합한 최고점 구절이 **클립보드에 자동 복사**되며 팝업이 닫힙니다. 그대로 카톡 창에 와서 **`Ctrl+V`**로 전송하면 끝입니다.

### 3. Multi-Clipboard 'Daily Devotional Cart' (v1.0.6)
- **English:** Standard operating systems can only hold one single text item in the clipboard memory at a time. BibleClip breaks this boundary by introducing a **Pre-set Verse Cart**. While doing your morning Quiet Time (QT) or reading scripture, you can tap the `➕` button to dump multiple powerful verses into your cart. Later, simply clicking an item in your cart instantly hot-swaps your current system clipboard memory with that specific verse, making it incredibly easy to gather and organize your favorite passages.
- **한국어 (나만의 말씀 장바구니 핫스왑):** 윈도우나 맥의 기본 클립보드는 한 번에 딱 하나의 텍스트만 저장할 수 있습니다. 다른 걸 복사하면 기존 데이터가 날아가는 한계를 깨기 위해 '말씀 장바구니'를 제공합니다. 아침 QT를 하거나 성경을 읽다가 마음에 와닿는 구절들을 장바구니에 `➕` 버튼으로 툭툭 담아둡니다. 나중에 수집된 구절들을 **마우스로 딸깍 클릭할 때마다 현재 내 PC의 클립보드가 해당 성구로 즉시 교체(Hot-Swap)**되어, 나만의 말씀 카드나 기도 노트를 만들 때 자유자재로 꺼내 쓸 수 있습니다.

### 4. Full-Screen Visual Meditation Mode (F11 Presentation)
- **English:** Pressing **`F11`** strips away all application controls, sliders, and sidebars, transforming the active scripture text into a clean, distraction-free Full-Screen Viewer. This is highly optimized for peaceful visual meditation, desktop wallpaper presentation, or direct video capture for Christian content creators.
- **한국어 (F11 집중 묵상 뷰어 모드):** 성구 카드에서 **`F11`**을 누르면 메뉴바, 사이드바, 조절 슬라이더 등 모든 잡다한 프로그램 UI가 사라지고, 오직 아름다운 말씀 텍스트만 화면에 가득 차는 전용 뷰어 모드로 진입합니다. 온전한 말씀 묵상에 집중하거나, 크리스천 콘텐츠 크리에이터들이 깨끗한 화면을 비디오로 캡처할 때 최적의 환경을 제공합니다.

## 🛠️ Technical Architecture & Developer Guide
### 🇰🇷 기술 아키텍처 및 개발자 가이드

The current production release is built on a high-performance **pywebview** frontend infrastructure (`bibleclip_web.py` + `web/`). The legacy CustomTkinter UI framework components remain preserved strictly for backward compatibility constraints.

현재 배포본은 **pywebview** 기반 웹 UI(`bibleclip_web.py` + `web/`)로 구성되어 있습니다. 과거 CustomTkinter UI 엔진 컴포넌트들은 하위 호환성 유지 목적을 위해 소스 내에 안전하게 보존되어 있습니다.

### ⚙️ How to Run (Development)

```bash
python bibleclip_web.py
# or
python -m bibleclip.webui
```

```
bibleclip_web.py        Web UI Application Entry Point → bibleclip.webui
bibleclip/              Core Application Package
├─ _version.py          Single-source of version truth (ASCII dedicated)
├─ config.py            Platform constants, paths, typography, and GitHub repository definitions
├─ constants.py         Character sets and alphabet-to-book mapping matrices
├─ text_utils.py        Korean string composition, purification, and search trigrams
├─ update.py            Release monitor engine & secure platform asset matching selectors
├─ data/                Engineered databases: bible_db.py & original_lang.py
├─ core/                Non-UI Library cores: parsers, formatters, and clipboard monitors
├─ webui/               Bridge controllers: api.py (JS-facing, pywebview independent)
└─ ui/                  (Legacy CustomTkinter package modules — backward compatibility only)

web/                    Frontend Distribution Layer (Production Workspace UI)
├─ index.html / app.js  Responsive layouts and runtime workspace lifecycle controllers
├─ css/                 Design tokens, application layouts, and localized typography stylesheets
└─ fonts/               Bundled cross-platform text runtimes (Pretendard)

bible_versions/         Target Bible SQLite Databases (Runtime assets fixed at root execution path)
original_lang/          Original language dictionaries & lexicons (Runtime assets fixed at root)
packaging/              Production packaging utilities: build_web.ps1 (Win) & build_mac.sh
tests/                  Headless regression unit tests (test_core, test_webui_api)
.github/workflows/      Automated cross-platform production release build pipelines
```
