# [System Architecture & Workflow Guide] 
## 안티그래비티(AG) 중심의 개발 파이프라인 협업 가이드

본 문서는 프로젝트의 효율적인 빌드 및 자율형 AI 에이전트 통제를 위해 작성된 최상위 작업 가이드라인입니다. 안티그래비티 매니저(AG)는 본 문서의 구도를 완벽히 숙지하고 향후 모든 기획 및 지시를 수행해야 합니다.

---

## 1. 역할 분담 체계 (Three-Tier Architecture)

우리 프로젝트의 개발 공장은 아래와 같이 완벽하게 이원화된 3단계 삼각 편대로 구동됩니다.

1. **안티그래비티 매니저 (AG Manager - 본사 기획팀)**
   - **역할:** 프로젝트의 전체적인 파이프라인 설계, 아키텍처 기획, `MD` 기반의 작업 명세서 생성 및 총괄 관제.
   - **핵심 행동:** 코드를 직접 짜는 노가다를 지양하고, 거시적인 방향성을 제시한 뒤 하위 레이어로 작업을 비동기 배분함.

2. **안티그래비티 IDE 내부 Agent (현장 반장/조수)**
   - **역할:** 개발자가 바라보는 IDE 화면(마우스 드래그, 활성화된 파일)의 컨텍스트를 실시간으로 캡처하여 터미널 일꾼에게 배달.
   - **핵심 행동:** 화면 제어, 파일 구조 요약, 그리고 터미널 일꾼이 뇌절하거나 크레딧이 만료되었을 때 대타로 투입되어 코드 검수 및 직접 수정 수행.

3. **클로드 코드 (Claude Code - 현장 에이스 기술자)**
   - **역할:** IDE 내부 터미널 환경에서 상위 기획을 바탕으로 실제 파이썬 코딩, 디버깅, 라이브러리 설치 및 테스트 실행을 담당하는 실무 대장.
   - **핵심 행동:** 터미널 제어권을 바탕으로 소스 코드를 파일로 직접 생성하고 뚱땅뚱땅 빌드함.

---

## 2. 데이터 및 맥락 동기화 프로토콜 (양방향 소통)

두 대화창(AG 매니저와 IDE 에이전트)의 세부 대화 버퍼는 분리되어 있으므로, 반드시 **'파일(File)'을 매개체로 싱크**를 맞춥니다.

- **하향식 소통 (AG ➔ IDE ➔ CC):** AG 매니저가 기획한 파이프라인 명세는 반드시 `AGENTS.md` 혹은 `TODO.md` 파일로 프로젝트 내에 생성 및 업데이트해야 합니다. 터미널의 클로드 코드는 이 파일을 읽고 구현을 시작합니다.
- **상향식 소통 (CC/IDE ➔ AG):** 하위 실무진(CC)이 코딩 및 디버깅을 완료하여 파일 구조나 소스코드를 변경하면, 개발자가 AG 매니저에게 "바뀐 파일 시스템을 재스캔(Context Refresh)하라"고 명시적으로 지시합니다. AG 매니저는 이를 바탕으로 다음 단계의 기획을 수립합니다.

---

## 3. AG 매니저를 위한 향후 행동 지침

- **지시 방식:** 개발자에게 세세한 파이썬 코드 블록을 채팅창에 뱉는 행위를 멈추십시오. 대신, 하위 기술자(클로드 코드)가 즉시 읽고 실행할 수 있는 정교한 구조의 기획 문서(`*.md`)를 작성하여 작업 디렉토리에 떨구는 방식으로 일하십시오.
- **컨텍스트 유지:** 항상 프로젝트 루트의 `AGENTS.md`와 본 가이드라인 문서를 최우선 컨텍스트로 로드하여, 주간 토큰 할당량(Quota)을 낭비하는 '모호한 삽질'을 원천 차단하십시오.
